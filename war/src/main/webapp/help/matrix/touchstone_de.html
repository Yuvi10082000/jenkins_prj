Ein "Touchstone" wird durch einen Kombinationsfilter definiert. Alle Builds, die diesem Filter entsprechen, werden zuerst ausgeführt. Nur wenn das Ergebnis die Ergebnis bedingung entspricht, werden die verbleibenden Builds ausgeführt.

Es ist auch möglich, mehrere Touchstones durch Semikolon getrennt zu definieren. Dies ist sinnvoll, wenn man mehrere Schritte (build, unit test, functional test) für mehrere Platformen ausführt, und sicherstellen möchte, dass ein Schritt für alle Platformen erfolgreich war, bevor man die Infrastruktur mit dem nächsten Schritt für irgendeine Platform belastet.

Beispiel:

Label expression PLATFORM = "win32 linuxX86"
Label expression SCHRITT = "build unit_test feature_test deploy"

hgTouchstone = SCHRITT="build"; SCHRITT="unit_test"; SCHRITT="feature_test"
