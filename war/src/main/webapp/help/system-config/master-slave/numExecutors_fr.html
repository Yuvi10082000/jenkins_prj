Contrôler le nombre de builds concurrents que Jenkins peut exécuter sur cet agent. La valeur touche la charge du système que Jenkins peut subir. Une bonne valeur pour commencer serait le nombre de processeurs.

Augmenter cette valeur au-delà du nombre de processeurs peut rallonger la durée individuelle des builds, mais aussi augmenter la capacité. En effet, cela permet au processeur de construire un projet pendant qu'un autre est en attente d'entrées/sorties.

Positionner cette valeur à 0 est utile pour rendre un agent temporairement indisponible, sans perdre toute la configuration.
