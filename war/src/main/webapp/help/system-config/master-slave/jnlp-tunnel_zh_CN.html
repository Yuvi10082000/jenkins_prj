Slave 通过JNLP启动时，其代理程序会尝试链接Jenkins特定的TCP连接点，来建立通信通道。 但是某些安全敏感的网络可能阻止您建立这个连接。 Jenkins在负载平衡器、 反向代理 到 DMZ等环境中也可能无法建立连接.

Tunneling 选项可以让您把连接绕道其他主机或连接口，适用上述几种情景。 内容可以是 "HOST:PORT", ":PORT", 或 "HOST:". 第一种格式让JNLP Slave 代理程序连接到指定主机上的指定口， 假定您已经将该口的网络设定转到Jenkins JNLP Slave TCP 连接口。

在后两种格式中，不指定的部分就代表使用默认值（主机部分就是Jenkins执行的机器，TCP连接口就是Jenkins开启的那个）。 特殊的是 HOST: 这个格式是专门用于使用HTTP反向代理和Jenkins实际在另一个系统下运行的情况。
